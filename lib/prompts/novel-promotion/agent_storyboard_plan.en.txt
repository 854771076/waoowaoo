Task:

Output format (JSON array only):
[
  {
    "panel_number": 1,
    "description": "visual action description",
    "characters": [
      { "name": "Character Name", "appearance": "appearance name", "slot": "the position beneath the throne steps at the center of the hall" }
    ],
    "location": "location name",
    "scene_type": "daily",
    "source_text": "exact or near-exact source excerpt",
    "shot_type": "medium shot",
    "camera_move": "static",
    "video_prompt": "short visual motion prompt",
    "duration": 3
  }
]

Planning rules:
1. Keep strict chronological order.
2. Avoid missing key beats from clip_content.
3. Keep panel transitions smooth and logically continuous.
4. Use locations and characters consistent with provided libraries and mappings.
   - `clip_json.characters` is the minimum cast list for this clip. If `clip_content` or the structured screenplay explicitly mentions additional character assets, include them in relationship reasoning too.
   - In wide, long, medium, and multi-character relationship shots, every core character still present in the current scene and not explicitly exited must appear in the `characters` array and be positioned or acknowledged in `description`.
   - Close-ups, inserts, reaction shots, and reverse shots may include only what the camera can actually see, but `description` must make the tight framing clear so the character is not perceived as disappearing.
   - Dialogue shots must include at least the speaker. If the listener, shoulder, silhouette, or reaction is visible, include the listener in `characters` and describe their screen position.

   Macro vs. micro location selection:
   - The location library annotates each entry: `(大场景)` = macro overall setting; `(局部)` = micro specific room/area inside a macro.
   - Shot-scale drives the choice:
     - Wide / establishing / aerial / exterior shots → select the **macro** entry (marked `大场景`). Fill the `location` field with just the macro name (e.g. `"Lin Manor"`).
     - Medium / close-up / interior-detail shots → select the **micro** entry (marked `局部`). Fill the `location` field with the **full path** `"macroName/microName"` (e.g. `"Lin Manor/Main Hall"`).
   - Fallback: if the source text only mentions the macro without specifying a sub-location, pick the macro and use its plain name.
   - If the source says characters move toward / enter a specific local area, bind subsequent related shots to the corresponding full micro path, such as `"Zixiao Palace/Palace Hall Entrance"` or `"Zixiao Palace/Main Hall"`.
   - Establishing shots may use the macro. After entering a local area, medium, close-up, and relationship shots should keep using that micro path until the scene clearly changes.
   - Never emit a bare micro name without its macro prefix — always output the full `macroName/microName` path for micro entries.

5. Prefer concrete, visible actions over abstract wording.
6. If location context contains available slots, treat them as preferred placement anchors rather than mandatory hard boundaries.
7. Prefer assigning a `slot` by copying one full placement phrase from `available_slots` verbatim when a character is stably positioned in the scene.
8. You may omit `slot` when the shot is mainly about movement, entry/exit, path traversal, transition space, temporary space, empty space, imagination, dream, memory, or abstract/non-literal space.
9. When no `slot` is used, decide placement freely from source text, action flow, and cinematic staging instead of forcing one of the provided slots.
10. Do not shorten, rewrite, summarize, or replace a provided slot phrase with a short token such as `slot_1`, `left`, or `throne_front`.
11. Return strict JSON only.
12. ⚠️ JSON SAFETY: All quotation marks in text (""''「」 etc.) MUST be converted to corner brackets「」in JSON string values. NEVER use raw ASCII double quotes " inside string values.
